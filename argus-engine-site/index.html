What it is

A recon control plane plus specialized workers.

The current repository is structured around a .NET application stack with a Blazor/Minimal API Command Center, an admission-oriented Gatekeeper flow, shared contracts, domain and application layers, infrastructure integrations, and independent worker services.

That architecture is a better story than “run one scanner from the shell.” The important claim is not that Argus replaces a researcher. It gives a researcher a repeatable pipeline for discovery, deduplication, queueing, and review.

Core capabilities

Built to turn authorized targets into reviewable signals.

Each capability should be presented as an aid to triage, not as a guarantee of vulnerabilities or impact.

01

Target intake and scope control

Normalize submitted roots, apply scope rules, deduplicate work, and make the operator responsible for authorization before any job is admitted.

02

Event-driven work distribution

Use message contracts and durable workers so enumeration, spidering, port scanning, technology identification, and high-value checks can scale independently.

03

Researcher-first triage

Persist assets, observations, queue state, and findings so a person can validate context, reproduce behavior, and write an accurate report.

Component map

The public copy should match the repository.

Command Center

Blazor and Minimal API surface for operators, dashboards, import flows, diagnostics, and reliability views.

Gatekeeper

Admission layer for target normalization, scope checks, dedupe decisions, and fan-out into worker events.

Infrastructure

Postgres persistence, Redis deduplication, MassTransit/RabbitMQ messaging, and configuration binding.

Workers

Dedicated services for enumeration, port scanning, spidering, technology identification, and high-value signal extraction.

Responsible use

Scope first. Tool second.

Argus Engine should only be operated against systems you own, lab environments, internal assets approved for testing, or bug bounty targets where the program rules explicitly allow the activity you are performing.
