what it is

Not a dashboard. A recon engine with teeth.

ArgusEngine is shaped like an operations platform for bug bounty and authorized reconnaissance: controlled intake, normalized targets, event-driven workers, durable queues, and audit-friendly state transitions.

01 // scoped intake

Targets enter through a gate, not a free-for-all.

The Command Center flow is designed around target creation, normalization, bulk import handling, and scope-aware orchestration before work hits the queue.

scope normalize dedupe
02 // worker mesh

Small workers. Clear jobs. Less chaos.

Enumeration, spidering, HTTP classification, technology identification, and port-scan orchestration live behind worker boundaries instead of being tangled together.

enum spider classify
03 // reliability

The platform remembers what happened.

Durable messages, queue transitions, outbox/inbox idempotency, and reliability baseline endpoints keep the system observable when scans get noisy.

outbox inbox slo

Why recon tools need admission control

A scanner without scope discipline is just liability with a progress bar. Gatekeeper-style intake makes authorization a first-class system concern.

recon scope
2026-05-01 // ops

Queues beat hero scripts

Scripts are fast to start and easy to lose control of. Queues give scans memory, backpressure, retry behavior, and a way to explain what happened later.

queues workers
2026-05-01 // .net

.NET is not boring. Bad demos are boring.

C#, Minimal APIs, EF Core, background workers, and message buses are a strong foundation for serious security tooling when the architecture is clean.
