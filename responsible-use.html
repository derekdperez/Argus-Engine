Responsible use policy

Argus Engine is for authorized security work only.

The project should be framed as tooling for disciplined researchers, internal teams, and lab environments—not as a shortcut for indiscriminate scanning.

Allowed use

Use the platform where permission is explicit.

Your own systems

Personal labs, owned domains, owned cloud accounts, and internal environments where you control the risk.

Approved internal testing

Corporate security work with written authorization, agreed scope, and clear escalation paths.

Bug bounty programs

Only within the exact scope, rate limits, and testing rules set by the program.

Training environments

CTFs, intentionally vulnerable labs, and local ranges designed for security practice.

Disallowed use

Do not create risk for others.

Do not run Argus Engine against random internet targets, third-party infrastructure without permission, excluded bug bounty assets, production systems where testing is not approved, or any environment where you are trying to evade detection, bypass rate limits, or hide attribution.

Do not treat observations as confirmed vulnerabilities. Report only what you can reproduce, explain, and responsibly disclose.
